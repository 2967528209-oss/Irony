# 引用追溯与来源标注

## 为什么需要引用

没有引用的 RAG 回答：
> "公司年假是 15 天。" → 用户：这可信吗？依据是什么？

有引用的 RAG 回答：
> "公司年假是 15 天。[来源：员工手册 v2.3, 第4章]" → 用户可验证

引用解决了：
- **可信度**：用户知道回答有据可查
- **可验证性**：用户可以去原文核实
- **责任归属**：区分 LLM 推理和事实陈述

## 实现方式

### 1. Prompt 指令法

在 System Prompt 中要求 LLM 标注来源：

```
基于以下资料回答用户问题。每个事实性陈述后，用 [来源X] 标注引用。

资料：
[来源1] 员工手册.pdf 第3页：年假为15天...
[来源2] HR政策.md：请假需提前3天...

用户问题：请假政策是什么？
```

LLM 输出：
```
公司规定年假为15天[来源1]，请假需提前3天在系统中申请[来源2]。
```

### 2. 后处理法

LLM 生成回答后，用文本匹配找出每句话来源于哪个文档块：

```
回答句子 → 与每个检索文档计算相似度 → 相似度最高的作为来源
```

### 3. 结构化输出法

让 LLM 直接返回结构化的引用数据：

```typescript
const schema = z.object({
  answer: z.string(),
  citations: z.array(z.object({
    text: z.string(),          // 引用的原文片段
    sourceId: z.string(),      // 来源文档 ID
    relevance: z.number(),     // 相关性评分
  })),
});
```

## 架构设计

```
检索结果（带元数据）
  ↓
为每个文档块分配编号：[来源1], [来源2], ...
  ↓
拼入 Prompt，要求 LLM 引用编号
  ↓
LLM 生成带引用标记的回答
  ↓
后端解析引用标记，关联到原文档的 metadata
  ↓
前端渲染：点击引用可跳转到原文
```

## 架构决策

| 决策 | 建议 | 原因 |
|------|------|------|
| 实现方式 | Prompt 指令法 + 结构化输出 | 效果好，可控性强 |
| 引用粒度 | 段落级 | 句子级太碎，文档级太粗 |
| 元数据 | 必须包含来源路径、页码、标题 | 支持前端跳转和溯源 |
