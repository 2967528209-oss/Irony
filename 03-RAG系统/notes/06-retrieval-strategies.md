# 检索策略

## 检索是 RAG 的核心瓶颈

RAG 系统中，LLM 只能基于检索到的内容生成回答。检索到的内容质量，决定了最终回答的上限。

```
检索到了正确文档 → LLM 大概率给出好回答
检索到了无关文档 → LLM 再强也无法给出正确回答
```

## 三种检索方式

### 1. 向量检索（Semantic Search）

```
用户问题 → Embedding → 与文档向量计算相似度 → 返回 Top-K
```

| 优点 | 缺点 |
|------|------|
| 理解语义（"汽车" 能匹配 "轿车"） | 对精确关键词不敏感 |
| 泛化能力强 | 对专有名词/编号匹配差 |

### 2. 关键词检索（Keyword / BM25）

```
用户问题 → 分词 → TF-IDF / BM25 算法 → 返回 Top-K
```

| 优点 | 缺点 |
|------|------|
| 精确匹配关键词 | 无法理解同义词 |
| 对专有名词/编号强 | "汽车" 无法匹配 "轿车" |
| 不需要 Embedding | 无语义理解 |

### 3. 混合检索（Hybrid Search）— 推荐

```
用户问题 ──→ 向量检索 → Top-K₁ ─┐
           │                     ├── 融合排序 → 最终 Top-K
           └→ 关键词检索 → Top-K₂ ─┘
```

两种检索互补：向量覆盖语义，关键词覆盖精确匹配。

### 融合算法：Reciprocal Rank Fusion (RRF)

```
RRF_score(d) = Σ 1 / (k + rank_i(d))

k = 60（常数）
rank_i(d) = 文档 d 在第 i 个检索结果中的排名
```

简单说：一个文档在多个检索结果中排名都靠前，融合后分数就高。

## MMR（最大边际相关性）

解决问题：Top-K 结果太相似，信息冗余。

```
普通 Top-5：文档A(95%), 文档A变体(93%), 文档A同义(91%), ...
→ 5 个结果说的是同一件事

MMR Top-5：文档A(95%), 文档B(85%), 文档C(80%), ...
→ 5 个结果覆盖不同角度
```

MMR 在选择下一个文档时，同时考虑**与问题的相关性**和**与已选文档的差异性**：

```
MMR = λ × 相关性(q, d) - (1-λ) × max(相似度(d, d_selected))

λ = 0.5: 平衡相关性和多样性
λ = 1.0: 纯相关性（退化为普通检索）
λ = 0.0: 纯多样性
```

## 架构决策

| 决策 | 建议 | 原因 |
|------|------|------|
| 检索方式 | 混合检索 | 语义 + 关键词互补 |
| Top-K 值 | 5-10 | 太少遗漏，太多引入噪音 |
| 是否用 MMR | 推荐 | 提高结果多样性 |
| 相似度阈值 | 0.7 | 过滤掉低相关结果 |
