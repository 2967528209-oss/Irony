# Embedding 模型

## 核心概念

Embedding 将文本映射为高维向量，语义相近的文本在向量空间中距离更近：

```
"猫" → [0.23, -0.15, 0.78, ...]    ┐
"小猫" → [0.25, -0.12, 0.80, ...]   ├ 距离近（语义相似）
"狗" → [0.20, -0.18, 0.72, ...]    ┘

"汽车" → [-0.45, 0.67, 0.12, ...]   ← 距离远（语义不同）
```

## 在 RAG 中的两个使用场景

1. **Indexing 时**：将文档块编码为向量，存入向量库
2. **Query 时**：将用户问题编码为向量，与文档向量做相似度匹配

关键：两个场景**必须用同一个 Embedding 模型**，否则向量空间不一致。

## 主流模型对比

| 模型 | 维度 | 上下文 | 多语言 | 特点 |
|------|------|--------|--------|------|
| OpenAI text-embedding-3-small | 1536 | 8191 | 是 | 性价比高 |
| OpenAI text-embedding-3-large | 3072 | 8191 | 是 | 质量更高 |
| BGE-M3 | 1024 | 8192 | 是 | 开源，支持稀疏+稠密 |
| Cohere embed-v3 | 1024 | 512 | 是 | 支持搜索/分类不同任务 |
| nomic-embed-text | 768 | 8192 | 是 | 开源，Ollama 可运行 |

## 相似度计算

| 算法 | 特点 | 适用 |
|------|------|------|
| 余弦相似度 | 方向相似度，忽略长度 | 最常用（默认选择） |
| 点积 | 考虑方向和长度 | 归一化后的向量 |
| 欧氏距离 | 绝对距离 | 较少使用 |

```
余弦相似度 = cos(θ) = (A · B) / (|A| × |B|)

范围：[-1, 1]
1 = 完全相同方向（最相似）
0 = 正交（无关）
-1 = 完全相反
```

## 架构决策

| 决策 | 建议 | 原因 |
|------|------|------|
| 模型选择 | 先用 OpenAI small，不够再升级 | 性价比优先 |
| 维度 | 1536 起步 | 太低丢精度，太高浪费存储 |
| 批量编码 | 一次发 100-500 条 | 减少 API 调用次数 |
| 缓存 | 相同文本不重复编码 | 幂等，节省成本 |
| 归一化 | 存入前归一化 | 方便后续用点积代替余弦 |
