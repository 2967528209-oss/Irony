# 阶段 5 架构总览：AI 应用架构模式

## 核心理解

阶段 5 的所有知识点回答一个问题：**把 AI 从"能跑"变成"能用"需要什么？**

```
阶段 2-4 解决的：怎么调 LLM、怎么做 RAG、怎么写 Agent
阶段 5 解决的：  怎么让这些东西在生产环境安全、可靠、可控地运行
```

## 全景架构

```
用户请求
  ↓
┌─ 接入层 ──────────────────────────────────────────────┐
│  协议转换 → 认证 → 限流 → 输入长度检查                    │
└──────────────────────────────────────────────────────┘
  ↓
┌─ 安全层 ──────────────────────────────────────────────┐
│  PII 脱敏 → Prompt Injection 检测 → 内容审核             │
└──────────────────────────────────────────────────────┘
  ↓
┌─ 编排层 ──────────────────────────────────────────────┐
│  Prompt 模板渲染 → 业务逻辑 (RAG/Agent/Chain)            │
│  → A/B 分流 → 异步任务分发                               │
└──────────────────────────────────────────────────────┘
  ↓
┌─ 模型层 ──────────────────────────────────────────────┐
│  模型路由 → 缓存检查 → LLM 调用 → 重试/降级              │
│  → Token 统计 → 调用日志                                │
└──────────────────────────────────────────────────────┘
  ↓
┌─ 数据层 ──────────────────────────────────────────────┐
│  向量库 / 关系库 / 缓存 / 文件存储                        │
└──────────────────────────────────────────────────────┘
  ↓
┌─ 输出层 ──────────────────────────────────────────────┐
│  结构化解析 → 输出审核 → PII 检查 → 返回用户              │
└──────────────────────────────────────────────────────┘
```

## 10 个知识点的关系

```
                    ┌── 成本优化（省钱）
                    │
分层架构（骨架）──→  ├── 安全防护（不出事）
                    │
                    ├── 可观测性（看得见）
                    │
                    └── 边缘场景（兜得住）

辅助机制：
  Prompt 管理     → 管理编排层的核心资产
  输出结构化       → 让 LLM 输出可程序化处理
  异步与队列       → 处理慢任务
  多租户与限流     → 支持多用户
  A/B 测试        → 用数据驱动优化
```

## 与 QQ Bot 项目的映射

你的 QQ Bot 可以对照这个架构做增强：

| 架构层 | 当前状态 | 可增强 |
|--------|---------|--------|
| 接入层 | ✅ NapCat WebSocket | 加输入长度限制 |
| 安全层 | ❌ 没有 | 加注入检测 + 内容审核 |
| 编排层 | ✅ 触发前缀 + 对话管理 | 加模型路由 (/ai vs /grok) |
| 模型层 | ✅ DeepSeek 调用 | 加重试、降级、缓存 |
| 数据层 | ✅ 内存对话历史 | 持久化到文件/数据库 |
| 输出层 | ❌ 没有 | 加输出审核 |
| 限流 | ❌ 没有 | 加用户频率限制 + 每日配额 |
| 可观测性 | ❌ 没有 | 加 Token 统计日志 |

## 实际建议

不是所有功能都要一开始就做。按优先级排序：

```
第一批（必须有）：
  限流 + 输入长度限制 + 基本错误处理
  → 防止被滥用和崩溃

第二批（应该有）：
  Token 统计 + 降级方案 + 结构化日志
  → 知道系统在干什么，出问题有兜底

第三批（锦上添花）：
  内容审核 + 缓存 + A/B 测试 + Prompt 管理
  → 优化体验和成本
```

## 一句话总结

**AI 应用的代码只占 20%，剩下 80% 是让它安全、可靠、可控地运行的工程。**
