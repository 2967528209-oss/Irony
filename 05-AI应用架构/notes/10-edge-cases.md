# 边缘场景处理

## AI 应用特有的"bug"

传统应用的 bug 通常是**确定性的**——相同输入总是产生相同错误。

AI 应用的问题是**概率性的**——有时候对，有时候错，有时候胡说八道。

## 三大边缘场景

### 1. 幻觉（Hallucination）

LLM 一本正经地编造不存在的信息：

```
用户: "张三丰是什么时候获得诺贝尔奖的？"
LLM:  "张三丰于 2015 年获得诺贝尔物理学奖，因为..."  ← 完全编造
```

**应对策略**：

| 策略 | 适用场景 |
|------|---------|
| RAG 检索 | 基于真实文档回答，减少编造 |
| 引用标注 | 要求标注信息来源（阶段 3 已实现） |
| Grounding 检查 | 检查回答是否有检索结果支撑 |
| 温度调低 | `temperature: 0` 减少随机性 |
| 置信度输出 | 让 LLM 评估自己的确信程度 |

```typescript
// 让 LLM 自我评估
const prompt = `回答以下问题。如果你不确定或没有相关信息，直接说"我不确定"。
在回答最后用 1-10 评估你的确信程度。

问题：${query}`;
```

### 2. 拒答策略

比幻觉更好的选择是**不回答**：

```
LLM 乱答（差）：
  "张三丰 2015 年获得诺贝尔奖..."

LLM 拒答（好）：
  "抱歉，我没有找到相关信息，建议查阅权威来源。"
```

```typescript
// System Prompt 中明确拒答规则
const systemPrompt = `回答规则：
1. 只基于提供的上下文回答
2. 如果上下文中没有相关信息，回复"根据现有资料，我无法回答这个问题"
3. 不要编造信息
4. 不确定的内容用"可能"、"据了解"等词修饰`;
```

### 3. 降级方案

当 LLM 服务出问题时的兜底：

```
                  正常 → DeepSeek API
                 /
请求 → 模型层 ──
                 \
                  异常 → 降级方案

降级策略：
  1. 备用模型：DeepSeek 挂了 → 切到本地 Ollama 模型
  2. 缓存兜底：返回相似问题的缓存答案
  3. 模板回复："服务繁忙，请稍后再试"
  4. 功能降级：复杂功能关闭，保留基本问答
```

```typescript
async function resilientChat(query: string): Promise<string> {
  try {
    return await deepseekClient.chat(query);
  } catch (err: any) {
    console.error('[Primary Failed]', err.message);

    // 降级 1：尝试备用模型
    try {
      return await backupClient.chat(query);
    } catch {
      // 降级 2：尝试缓存
      const cached = cache.getSimilar(query);
      if (cached) return `[缓存回答] ${cached}`;

      // 降级 3：模板回复
      return '服务暂时不可用，请稍后再试~';
    }
  }
}
```

## 其他边缘场景

| 场景 | 问题 | 应对 |
|------|------|------|
| 超长输入 | 用户贴了一整篇文章 | 截断 + 提示"输入过长" |
| 多语言混合 | 中英文混杂导致理解偏差 | Prompt 中指定语言规则 |
| 格式要求 | 用户要表格但 QQ 不支持 | 检测平台能力，调整输出格式 |
| 持续调戏 | 用户反复测试 LLM 边界 | 检测重复模式，触发拒答 |
| 上下文丢失 | 长对话后 LLM 忘了前面的内容 | 记忆压缩（已实现） |

## 容错设计清单

给任何 AI 应用做容错，检查以下几项：

| 检查项 | 问自己 |
|--------|--------|
| LLM API 挂了怎么办 | 有备用模型/缓存兜底吗 |
| 回答错了怎么办 | 有引用标注/置信度吗 |
| 输入恶意内容怎么办 | 有注入检测/内容审核吗 |
| 用户刷量怎么办 | 有限流/配额吗 |
| Token 超预算怎么办 | 有预算告警吗 |
| 回答包含敏感信息怎么办 | 有 PII 过滤/输出审核吗 |

## 核心原则

**AI 应用永远不能 100% 信任 LLM 的输出。** 把 LLM 当成一个「有能力但不完全可靠的同事」——它能做很多事，但你需要检查它的工作。
